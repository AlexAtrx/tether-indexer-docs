For this ticket, I shared what I thought is my finding: 

---
This looks like deploy/restart fallout, not a Hyperswarm root cause.

During the shard PM2 restart, old shard proc workers were still running syncWalletTransfers while shutdown had already started closing Mongo and RPC resources.

This made checkpoint reads fail with MongoNotConnectedError and running RPC calls fail with logs ERR_WALLET_TRANSFER_RPC_FAIL.

The fix is to make shard shutdown job-aware.

**Keys to add**

1. **direction** – indicates whether the transaction is “sent”, “received”, “swap_out” or “swap_in”. It must be a required field in the transaction object.  
2. **totalAmount** – a computed field that represents the net impact on the user’s balance: for outgoing transactions it’s amount + network fee, for incoming it’s just amount.  

**Where to add them**

- Inside the **Transaction object** of the **Transaction History response schema**.  
- Update the DTO / model that builds the JSON payload returned by the Transaction History endpoint.  
- Reflect the new fields in the OpenAPI (or Swagger) definition for the endpoint, marking *direction* as required and *totalAmount* as read‑only/computed.  

**Which API**

- The **Transaction History API** (the GET endpoint that returns the list of transactions, e.g. `GET /api/v1/transactions` or its equivalent in the service).  

These two fields should be added to the response schema of that endpoint, and the PR that implements the changes should be pushed for review.

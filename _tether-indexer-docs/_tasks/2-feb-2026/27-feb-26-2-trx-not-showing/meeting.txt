The issue discussed is that on **staging**, the **transaction history API** is outdated by about **10 days**. The **indexer** is correctly syncing new Bitcoin transactions to the database, but the **data shard and API layer** are not reflecting those updates. This means the backend services can’t access the latest indexed data, so users only see old transactions. Patricia and Gohar confirmed it’s not limited to Bitcoin — **USDT and cash-out transactions** are also missing — implying the problem is **systemic**, likely in the **transfer processor or data propagation path** between the indexer and shard API, not chain-specific.

:red_circle: HyperDHT/UDX Cross-Container RPC Failure — macOS Docker Desktop

  Every cross-container @hyperswarm/rpc request fails. Same-container (localhost) works fine.

  Failing paths:

  • rumble-app-node → rumble-ork (connect, addWallet, storeDevice) → CHANNEL_CLOSED / RPC client closed
  • rumble-ork → rumble-data-shard-api (addWallet forwarded) → CHANNEL_CLOSED
  • evm-indexer-api → evm-indexer-proc → CHANNEL_CLOSED (never triggered in practice — MongoDB mode skips RPC)

  Working:
  • rumble-data-shard-proc → itself (localhost) → :white_check_mark: Works

  Root cause: Docker Desktop on macOS runs containers in a Linux VM with virtio networking. UDX (UDP transport used by
  HyperDHT) data streams fail between containers — the Noise handshake completes (relayed through DHT bootstrap) but no
  data flows. The server never receives the connection.

  What works vs what breaks:

  :white_check_mark: Raw UDP (dgram) — cross-container works
  :white_check_mark: DHT peer lookup — works
  :white_check_mark: Noise handshake — appears to complete (relayed via bootstrap)
  :x: UDX data transfer — fails (server never receives connection)
  :x: @hyperswarm/rpc — CHANNEL_CLOSED / CHANNEL_DESTROYED

  Tested with: udx-native 1.19.2, hyperdht 6.29.0, @hyperswarm/rpc 3.4.1, node:22-bookworm + node:22-bookworm-slim

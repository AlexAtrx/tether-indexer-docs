# Description

The AutobaseLookupStorage router registers handlers for the @wdk-ork/save-wallet-id-lookups-batch and @wdk-ork/delete-lookup routes, but the hyperdispatch build script never declared them, so the autogenerated spec files were missing these two routes. At worker startup the Router.add() call threw:

    Cannot register a handler for a nonexistent route:
    @wdk-ork/save-wallet-id-lookups-batch


preventing the ork from starting under lookupEngine=autobase.
Declare both routes (and their request schemas) in build.js and regenerate spec/hyperschema, spec/hyperdispatch and spec/hyperdb so the on-disk spec matches the code. New routes are appended to preserve existing dispatch IDs/versions for already-persisted autobase data.
These changes were added in the following commits:
    batch-lookups change: https://github.com/tetherto/wdk-ork-wrk/commit/bdb9d43
    delete lookup change: https://github.com/tetherto/wdk-ork-wrk/commit/ba0fe10
